== Qanday Hissa Qo'shish Mumkin

=== Xatoliklar Haqida Xabar Berish

Agar xatolik topsangiz:

. https://github.com/yourusername/beb-fragrance/issues[Issues] bo'limiga o'ting
. Yangi issue yarating
. Quyidagi ma'lumotlarni kiriting:
  - Xatolikning qisqacha tavsifi
  - Xatolikni qanday takrorlash mumkinligi (qadam-baqadam)
  - Kutilgan natija va haqiqiy natija
  - Skrinshotlar (agar mavjud bo'lsa)
  - Muhit ma'lumotlari (brauzer, OS, Node.js versiyasi)

=== Yangi Xususiyat Taklif Qilish

. Issues bo'limida yangi feature request yarating
. Taklifingizni batafsil tavsiflang
. Nima uchun bu xususiyat kerakligini tushuntiring
. Implementatsiya bo'yicha takliflaringiz bo'lsa, baham ko'ring

=== Kod Hissa Qo'shish

==== 1. Repositoriyani Tayyorlash

[source,bash]
----
# Repositoriyani fork qiling (GitHub interfeysi orqali)

# Fork qilingan repositoriyani klonlang
git clone https://github.com/YOUR-USERNAME/beb-fragrance.git
cd beb-fragrance

# Asosiy repositoriyani upstream sifatida qo'shing
git remote add upstream https://github.com/yourusername/beb-fragrance.git

# Packagelarni o'rnating
npm install

# .env.local faylini yarating
cp .env.example .env.local
# Kerakli o'zgaruvchilarni to'ldiring
----

==== 2. Yangi Branch Yaratish

[source,bash]
----
# Asosiy branch'ni yangilang
git checkout main
git pull upstream main

# Yangi branch yarating
git checkout -b feature/your-feature-name
# yoki
git checkout -b fix/your-bug-fix
# yoki
git checkout -b docs/your-doc-update
----

Branch nomlash qoidalari:
- `feature/` - yangi xususiyatlar
- `fix/` - xatolik tuzatishlar
- `docs/` - hujjatlashtirish
- `refactor/` - kod qayta tashkil qilish
- `style/` - formatlash, stillash
- `test/` - test qo'shish

==== 3. Kod Yozish

*Kod Standartlari:*

- TypeScript ishlating
- Server Component bo'la oladigan joyda `"use client"` ishlatmang
- Har bir komponent uchun TypeScript interfeyslari yozing
- Zod validatsiya ishlating (API routelar uchun)
- Tailwind CSS klasslari, `cn()` utility ishlating
- Loading, error, empty state'lar qo'shing
- Responsive dizayn (mobile-first)

==== 4. O'zgarishlarni Commit Qilish

[source,bash]
----
# O'zgarishlarni stage qilish
git add .

# Commit yozish
git commit -m "feat: add product filtering by category"

# yoki
git commit -m "fix: resolve cart quantity update bug"

# yoki
git commit -m "docs: update installation instructions"
----

Commit xabar formatlari:
- `feat:` - yangi xususiyat
- `fix:` - xatolik tuzatish
- `docs:` - hujjatlashtirish
- `style:` - formatlash
- `refactor:` - kod qayta tashkil qilish
- `test:` - test qo'shish
- `chore:` - texnik xizmat

==== 5. Push va Pull Request

[source,bash]
----
# Branch'ni push qiling
git push origin feature/your-feature-name

# GitHub'da Pull Request yarating
----

Pull Request tavsifi:
- Nima o'zgarganligini yozing
- Nima uchun bu o'zgarish kerakligini tushuntiring
- Skrinshotlar (agar UI o'zgargan bo'lsa)
- Test qilinganligini tasdiqlang

== Loyiha Haqida

*Beb Fragrance* - bu premium parfyumeriya mahsulotlarini sotish uchun yaratilgan zamonaviy, to'liq ishlaydigan e-commerce platforma. Loyiha real pul ishlaydigan, foydalanuvchilar Google orqali kiradigan, mahsulotlarni ko'rib savatga soladigan va haqiqiy karta orqali to'lov qiladigan to'liq online do'kon.

== O'rnatish va Ishga Tushirish

=== Talablar

- *Node.js* 18.17 yoki undan yuqori
- *npm* 9.0 yoki undan yuqori (yoki *yarn*/*pnpm*)
- *PostgreSQL* ma'lumotlar bazasi (Neon tavsiya etiladi)
- *Stripe* hisobi (test yoki production)
- *Google Cloud* hisobi (OAuth uchun)
- *Resend* hisobi (email xabarlar uchun)

=== 1. Repositoriyani Klonlash

[source,bash]
----
git clone https://github.com/yourusername/beb-fragrance.git
cd beb-fragrance
----

=== 2. Packagelarni O'rnatish

[source,bash]
----
npm install
----

=== 3. Environment Variables Sozlash

`.env.example` faylidan `.env.local` ga nusxa oling:

[source,bash]
----
cp .env.example .env.local
----

`.env.local` faylini oching va quyidagi o'zgaruvchilarni to'ldiring:

[source,env]
----

DIRECT_URL="postgresql://neondb_owner:[YOUR-PASSWORD]@ep-example.us-east-1.aws.neon.tech/neondb?sslmode=require&channel_binding=require"

# NextAuth

NEXTAUTH_SECRET="your-secret-key-minimum-32-characters"

# Stripe
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY="pk_test_your-publishable-key"

== Xizmatlar Sozlamalari

=== Neon Sozlamalari

. https://neon.tech[neon.tech] saytida ro'yxatdan o'ting
. Yangi loyiha yarating
. *Connection Details* bo'limidan `DATABASE_URL` va `DIRECT_URL` ni oling
. `DATABASE_URL` uchun pooler string ishlating
. `DIRECT_URL` uchun migratsiya va schema push uchun direct string ishlating

=== Supabase Storage Sozlamalari

. https://supabase.com[supabase.com] saytida ro'yxatdan o'ting
. Yangi loyiha yarating
. *Settings > API* bo'limidan API kalitlarini oling
. *Storage* bo'limida `products` va `categories` bucket'larini yarating
. Bucket'lar uchun *public* ruxsatini yoqing

=== Google OAuth Sozlamalari

. https://console.cloud.google.com[Google Cloud Console] saytiga kiring
. Yangi loyiha yarating
. *APIs & Services > Credentials* bo'limiga o'ting
. *OAuth 2.0 Client ID* yarating
. *Authorized redirect URIs* ga qo'shing:
  - `http://localhost:3000/api/auth/callback/google`
  - `https://yourdomain.vercel.app/api/auth/callback/google`
. Client ID va Client Secret ni `.env.local` ga ko'chiring

=== Stripe Sozlamalari

. https://stripe.com[stripe.com] saytida ro'yxatdan o'ting
. *Developers > API keys* bo'limidan kalitlarni oling
. *Webhooks* bo'limida endpoint yarating:
  - URL: `https://yourdomain.vercel.app/api/webhooks/stripe`
  - Events: `checkout.session.completed`
. Webhook secret ni `.env.local` ga ko'chiring

=== Resend Sozlamalari

. https://resend.com[resend.com] saytida ro'yxatdan o'ting
. API kalitini oling
. Domain'ingizni verify qiling
. `.env.local` ga API kalitini qo'shing

== Tez-tez So'raladigan Savollar

=== Q: Loyiha ishga tushmayapti?

*A:* `.env.local` faylidagi barcha o'zgaruvchilar to'g'ri ekanligini tekshiring. 
`npx prisma migrate dev` va `npx prisma generate` komandalarini ishga tushiring.

=== Q: Google login ishlamayapti?

*A:* Google Cloud Console'da `redirect_uri` to'g'ri ekanligini tekshiring. 
Localhost va production URL'lar ikkalasi ham qo'shilgan bo'lishi kerak.

=== Q: To'lov ishlamayapti?

*A:* Stripe test rejimida ekanligingizni va test karta ma'lumotlarini 
ishlatayotganingizni tekshiring. Webhook endpoint URL'ni to'g'ri sozlang.

=== Q: Rasmlar yuklanmayapti?

*A:* Supabase Storage bucket'larini yaratganingizni va ularga *public* ruxsat 
berganingizni tekshiring.

=== Q: Admin panelga kira olmayapman?

*A:* Foydalanuvchi rolingiz `ADMIN` ekanligini tekshiring. 
Prisma Studio orqali `User` jadvalidan rolingizni tekshiring.
